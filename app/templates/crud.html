CRUD

What is CRUD?

CRUD is an acronym for Create, Read, Update and Delete, and refers to four functions that are necessary for implementing a persistent storage application and are inherent for relational databases, persistent storage refers to data storage that still exists even after the device was powered off.

CREATE

The create function allows us to make new records to store in a database. Records in relational databases are represented as rows in a table.

In SQL we create new records by using INSERT, declaring the table in which we're inserting and defining the VALUES of the new record.

INSERT INTO client (id, first_name, last_name, age) VALUES (1, "Jorge", "Tostado", 21);

READ

The read function is searching for specific records inside the database, we can filter using certain criteria and show only specific data of the record through the query.

We read by using SELECT, specifying the data to show by naming it (if all then we use *), naming the table we're searching and finally naming the criteria we're looking for.

SELECT * FROM client WHERE id=1;

UPDATE

The update function is changing the values of existing records within the database.

For updating we use UPDATE in the table of the record we're updating, SET the new values of the record WHERE the criteria matches.

UPDATE TABLE client SET age=22 WHERE id=1;

DELETE

The delete function means erasing records from our database, some databases perform a hard delete which erases all data and no longer retrievable, while other perform a soft delete which removes the record from the table but keeps the data present and intact.

Use DELETE on the table we want to delete from and VERY IMPORTANT to specify the criteria of the records we want to delete.

DELETE FROM client WHERE id=1;

SCAN

The scan function refers to looking through all records of a table in the database, much like READ but we don't specify criteria to filter through.

SELECT * FROM client;
